# Running Jietu v2.0 软件说明书

## 概述

这是一个极简托盘截图工具，主要功能是：
- 区域截图
- 将截图钉在屏幕最前面（置顶贴图）
- 调用 RunningHub AI 进行图片处理
- 历史记录管理

---

## 核心文件说明

### 1. 主程序文件

| 文件 | 作用 |
|------|------|
| main.js | 软件的核心大脑，负责所有功能的实现，包括窗口管理、截图、置顶图、RunningHub API 调用等。软件运行时它会一直在后台运行。 |
| preload.js | 桥梁文件，连接主程序和网页界面，让网页可以调用系统功能（如读写文件、剪贴板等）。 |

### 2. 界面文件

| 文件 | 作用 |
|------|------|
| capture.html / capture.js | 区域截图的界面。截图时弹出的半透明窗口，让你用鼠标选择截图区域。 |
| pin.html / pin.js | 置顶贴图的界面。每个钉在屏幕上的图片都是一个独立的窗口，这个文件定义了它们的外观和行为。 |
| history.html / history.js | 历史记录界面。查看和管理之前截过的图片。 |
| settings.html / settings.js | 设置界面。修改快捷键、API 密钥等配置。 |
| workflow-selector.html / workflow-selector.js | 工作流选择界面。管理和选择要使用的 AI 工作流。 |

### 3. 配置文件

| 文件 | 作用 |
|------|------|
| runninghub.config.json | 软件的配置文件。保存了 API 密钥、快捷键设置、默认工作流等。所有设置都会保存在这个文件里。 |
| package.json | 项目的身份证，记录了软件名称、版本号、依赖库等信息。 |

### 4. 工作流文件

| 文件 | 作用 |
|------|------|
| runninghub-workflows/*.json | RunningHub 的工作流配置文件。每个文件代表一个 AI 图片处理流程（如图生图、换脸等）。 |
| runninghub-workflows/*.png | 工作流的缩略图，显示在选择界面中方便辨认。 |

## 功能操作说明

### 1. 截图
- 方式一：右键托盘图标 → 点击区域截图
- 方式二：使用快捷键 Ctrl+Alt+X（可自定义）

### 2. 置顶贴图
置顶贴图是钉在屏幕最前面的图片窗口，具有以下功能：
- 重新截图
- 保存
- 关闭
- 右键菜单上传到 RunningHub AI 处理

### 3. 粘贴为置顶贴图
- 方式一：右键托盘 → 点击粘贴为置顶贴图
- 方式二：快捷键 Ctrl+Alt+V（可自定义）

### 4. RunningHub AI 处理
1. 在置顶图上右键
2. 选择上传到 RunningHub 生图
3. 等待 AI 处理完成

### 5. 工作流管理
右键托盘 → 点击管理工作流，可以导入、编辑、删除工作流

### 6. 历史记录
- 右键托盘 → 点击打开历史记录
- 单击选中，双击打开
- 自动保留最近 50 组记录

### 7. 设置
右键托盘 → 点击设置，可以修改快捷键、API 密钥等

## 常见问题

Q: 软件打不开怎么办？
A: 确保已安装依赖，运行 npm install 后再试。

Q: 快捷键不生效？
A: 检查设置中快捷键是否正确保存。

Q: RunningHub API 调用失败？
A: 检查 apiKey 是否正确，网络是否正常。
